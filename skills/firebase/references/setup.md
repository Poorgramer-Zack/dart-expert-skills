# 01 - Firebase Core Setup (`firebase_core`)

## Goal
Initialize the Firebase SDK reliably across all Flutter platforms. This chapter focuses on eliminating manual native configuration files by enforcing the usage of `flutterfire_cli`, ensuring a seamless and deterministic setup process.

## Instructions

### 1. Mandatory CLI Initialization
Never manually download or move `GoogleService-Info.plist` (iOS) or `google-services.json` (Android). They do not configure Dart-only platforms (Web, macOS, Windows) and frequently break native build processes.

**Rule**: Always use the `flutterfire_cli`.
```bash
# 1. Activate the CLI globally
dart pub global activate flutterfire_cli

# 2. Run configure in the project root
flutterfire configure --project=your-firebase-project-id
```
This generates `lib/firebase_options.dart` containing all platform-specific keys natively injected into Dart.

### 2. Dart Initialization Sequence
Initialize Firebase optimally inside `main.dart` *before* `runApp`.

```dart
import 'package:firebase_core/firebase_core.dart';
import 'package:flutter/material.dart';
import 'firebase_options.dart'; // Auto-generated by flutterfire_cli

void main() async {
  // CRITICAL: Ensure native bindings are initialized before Firebase
  WidgetsFlutterBinding.ensureInitialized();

  // Initialize Firebase purely through Dart
  await Firebase.initializeApp(
    options: DefaultFirebaseOptions.currentPlatform,
  );

  runApp(const MyApp());
}
```

### 3. iOS/macOS Specific Constraints (CRITICAL)

When building for Apple platforms, the massive number of Firebase dependencies can frequently break the CocoaPods build process due to deployment target mismatches.

**Rule (`ios/Podfile` & `macos/Podfile`)**: 
1. The global platform target MUST be explicitly set (usually iOS 13.0+ or macOS 10.15+).
2. You MUST utilize static frameworks by uncommenting or adding `use_frameworks! :linkage => :static`. Dynamic linkage drastically slows down app startup time with Firebase.

```ruby
# Uncomment this line to define a global platform for your project
platform :ios, '13.0' # Ensure this matches your project requirements

target 'Runner' do
  # CRITICAL for Firebase:
  use_frameworks! :linkage => :static

  # ...
end
```

## Constraints
*   **Version Alignments:** Ensure that the `firebase_core` package version aligns with the other specific Firebase packages (like `firebase_auth`, `cloud_firestore`) you are installing. Use the `firebase_core` changelog to determine breaking cross-package changes.
